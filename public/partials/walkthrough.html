Applications

In order to start using the Teech.io backend, the first thing you need to do is wiring up your application to your Teech.io backend. It's very simple, just add the application from your Teech.io dashboard. Each application has an ID that you need to use together with your API key in order to authenticate your application when requesting data from the backend.

API

The REST API allows an app to interact with its backend on Teech.io programmatically via simple HTTP requests. The API is accessed via the http://api.teech.io domain.

JSON will be returned in all responses, including errors. The HTTP response codes are used to indicate the status of each request.

Authentication

All requests to teech.io REST API require you to authenticate via HTTP headers. The Teech-Application-Id identifies the application you’re accessing and the Teech-REST-API-key authenticates your request.

Collections

Collections are grouping the objects within Teech.io backend. Collections are like tables in relational databases.

Next steps will give you an overview of how collections work and of the main collection types that you can use.

next

Key/Value

Key/Values pairs are the way you can define properties for each object stored in a collection.

  • GET
  • POST
  • PUT
  • DELETE
 
curl http://api.teech.io/users \
 -X GET
 
curl http://api.teech.io/users \
 -X POST \
 -H 'Content-Type: application/json' \
 -d '{"username":"tim", "password":"Kld98Sld9", "name":"Tim Red"}'
 
curl http://api.teech.io/users \
 -X PUT \
 -H 'Content-Type: application/json' \
 -d '{"fullname":"Tim Roth"}'
 
curl http://api.teech.io/users/8d93iI093jm \
 -X DELETE
next

Queries

Queries let you search through a collection and limit and/or skip the results.

  • GET
 
curl http://api.teech.io/users?query={"username":"tim"} \
 -X GET
